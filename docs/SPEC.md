# SPEC（動画仕様）

## 対象

企画「ぼくが考えた変形ロボ」のショート動画。1本＝1つのお題（動物・植物・建物・食べ物・日用品など）。

## 動画フォーマット

| 項目 | 内容 |
|---|---|
| 尺 | 15〜20秒 |
| 画角 | 縦型（9:16 想定、1080×1920） |
| 出力形式 | mp4 |
| 撮影・顔出し | なし |
| 音声 | AIナレーション（言語: 日本語） |

## 構成要素

1. **お題提示**: 題材（例: 「掃除機」）をテロップ＋ナレーションで見せる
2. **変形前ビジュアル**: 題材そのもののAI静止画
3. **変形演出**: 「変形ロボ化」の見せ場。AI動画生成（image-to-video、後述）で実際に動く演出にする＋テロップ＋効果音で盛り上げる（2026-07-15変更。詳細は下記「変形・決めポーズシーンのAI動画生成」参照）
4. **変形後ロボ披露**: オリジナル変形ロボの見せ場。こちらもAI動画生成（image-to-video）で動きを出す（2026-07-15変更。以前は静止画のみだったが、決めポーズにも動きを持たせる方針に変更）
5. **一言オチ/コメント**: 図鑑的な一言、または妄想エンタメ的なコメント（啓発・注意喚起トーンにしない）

## 使用素材の組み合わせ方針

- **AI背景画像**: 「変形」「決めポーズ」以外の3シーン（お題提示・変形前・オチ）は静止画を用意し、軽いズーム（Ken Burns風）や切り替えで動きを出す
- **AI動画（変形・決めポーズシーン）**: Flux.2 Klein 9Bで生成した「変形前」「変形後」の2枚の静止画をそれぞれ起点に、image-to-video（Wan 2.2 I2V 14B FP8）で「変形」シーン（4秒程度想定）と「決めポーズ」シーンの動画を作る（2026-07-15変更。詳細は下記「変形・決めポーズシーンのAI動画生成」参照）。具体的な生成時間・コストは未検証
- **AI音声**: ナレーション用TTS。エンジンは未定（`docs/REPO_RESEARCH.md` の比較を踏まえて選定、Codexレビューで確定させる）
- **テロップ**: ナレーションに合わせて表示。タイミング取得方法（自動 or 手動）は未定
- **効果音**: `assets/sfx/` から選択して合成（変形音、決めポーズ音など）
- **合成**: FFmpegベースを想定（静止画のズーム=zoompanフィルタ、切り替え=xfade等、変形シーンは生成済み動画クリップをそのまま合成）。Docker必須・重量ローカルモデル前提の構成は避ける（`docs/REPO_RESEARCH.md` 参照）

## 変形・決めポーズシーンのAI動画生成（未検証・要調査）

2026-07-14、ユーザーからのフィードバックにより「変形シーンは動画で見せたい（高品質にしたい）」という要望が判明し、当初の「いきなり重い動画生成AIは使わない」方針から、変形シーンに限定してAI動画生成を使う方針に変更した。同日、初期採用候補のLoRA（WAN 2.1 14B T2V向け）がtext-to-video専用と判明したため、一時的にtext-to-video前提に変更していた。

2026-07-15、上記text-to-video前提を撤回し、以下の方式に変更した。

**変更の経緯**：「テレビ→ロボ」のような機械的変形（部品が伸縮・回転・分割結合する動き）は、Wan 2.2のようなimage-to-video系モデルが学習していない動きであり、LoRA（Civitai上に類似LoRAはあるが機械的変形specificなものは未確認）なしでは再現困難と判断した。そのため「機械的変形の過程そのものをAIに描かせる」ことをやめ、「同じ形状のまま動く動き（AIが得意な領域）」＋「編集（フラッシュカット）による変形の"隠し"」の組み合わせに設計を変更した。

**方式**：
1. Flux.2 Klein 9Bで「変形前」「変形後」の静止画2枚を生成（既存ワークフロー`flux2_klein_bokurobo_beforeafter_v2.json`で対応済み）
2. 「変形前」画像を起点に、image-to-video（Wan 2.2 I2V 14B FP8）で"溜め"の動き（振動・発光、**形状は変化させない**）を生成 → `transform`シーンのvideo（4秒程度）。末尾に数フレームの白フラッシュを合成し、次シーンへのハードカットの継ぎ目を隠す
3. 「変形後」画像を起点に、image-to-video（Wan 2.2 I2V 14B FP8）で"決めポーズ"の動き（振り向き・ポーズ、**パーツ構成は固定**、姿勢と勢いのみ変化）を生成 → `after`シーンのvideo
4. フラッシュ効果は各クリップ内に焼き込む形で実現し、`episode.json`の`transition`（`cut`/`zoom`の2値、`docs/RENDER_DESIGN.md`参照）やシーン間の`filter_complex`合成には手を加えない。シーン単位で確定済みmp4を`-c copy`で結合する既存のrender設計を維持する

**未検証事項**：
- Wan 2.2 I2V 14B FP8での実際の生成時間・VRAM・RunPodでのコスト試算（1クリップ約12分という実績値はあるが、本ユースケースでの再現性は未検証）
- 「変形前」「変形後」2枚の画像の背景・カメラ一致精度（`flux2_klein_bokurobo_beforeafter_v2.json`のSeed固定＋文言共通による担保。フラッシュカットで多少のズレは吸収できる想定だが未検証）
- 存在確認できないモデル名・ノード名は断定しない。本実装に入る前に調査・Codexレビューを行う
- RunPodのNetwork Volume 0GB運用（セッション毎に環境を再構築する前提）と、動画生成モデルのモデルサイズ・DL時間の相性

## 表現面のルール

- 「トランスフォーマー」という名称を使わない
- 既存作品・既存キャラクターのデザイン・名称に寄せない。オリジナル変形ロボとして扱う
- 啓発系・注意喚起系のトーンにしない。エンタメ／妄想／図鑑／コメントお題型にする

## スコープ外（今回は扱わない）

- SNS自動投稿（後回し）
- 「変形」シーン以外へのtext-to-video/image-to-video動画生成の適用（他4シーンは引き続き静止画のみ）
- 実写・顔出し素材

## 決定事項（2026-07-13 Codexレビュー反映、2026-07-15追記）

- テロップのタイミング取得方法: 初版は`episode.json`のシーン単位で手動秒指定とする（ASRによる自動抽出は将来差し替え候補）
- 機械可読データは`episodes/<話数>/episode.json`（標準JSON、追加ライブラリ不要）。人間向けの`plan.md`と併用する
- パイプラインは`validate`（マニフェスト検証）→`render`（動画合成）の2段構成とし、両サブコマンドを実装済み
- AI画像生成の実行手段：既存のComfyUI/Flux.2 Klein 9B環境を流用する（`flux2_klein_bokurobo_beforeafter_v2.json`）
- 変形・決めポーズシーンの動画生成方式：image-to-video（Wan 2.2 I2V 14B FP8）＋フラッシュカット編集。詳細は上記「変形・決めポーズシーンのAI動画生成」参照（2026-07-15決定、text-to-video前提から変更）

## 未決定事項（実装前に確定させる）

- TTSエンジンの選定: Piperは日本語voice未提供＋GPLv3のため既定候補から除外。edge-ttsはオンライン依存のためoptional provider扱い。当面は事前生成済み音声ファイルを入力とする（詳細は`docs/CODEX_REVIEW_RULES.md`参照）
- 1本あたりのカット数・画像枚数の標準値（現状は5シーン=5カット想定）
- Wan 2.2 I2V 14B FP8の実行環境・VRAM要件・生成時間・コスト試算（未検証。上記「変形・決めポーズシーンのAI動画生成」参照）
