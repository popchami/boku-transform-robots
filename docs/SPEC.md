# SPEC（動画仕様）

## 対象

企画「ぼくが考えた変形ロボ」のショート動画。1本＝1つのお題（動物・植物・建物・食べ物・日用品など）。

## 動画フォーマット

| 項目 | 内容 |
|---|---|
| 尺 | 15〜20秒 |
| 画角 | 縦型（9:16 想定、1080×1920） |
| 出力形式 | mp4 |
| 撮影・顔出し | なし |
| 音声 | AIナレーション（言語: 日本語） |

## 構成要素

1. **お題提示**: 題材（例: 「掃除機」）をテロップ＋ナレーションで見せる
2. **変形前ビジュアル**: 題材そのもののAI静止画
3. **変形演出**: 「変形ロボ化」の見せ場。AI動画生成（image-to-video）で実際に動く演出にする＋テロップ＋効果音で盛り上げる（2026-07-14変更。他4シーンとは異なり、このシーンのみ動画生成AIを使う）
4. **変形後ロボ披露**: オリジナル変形ロボのAI静止画
5. **一言オチ/コメント**: 図鑑的な一言、または妄想エンタメ的なコメント（啓発・注意喚起トーンにしない）

## 使用素材の組み合わせ方針

- **AI背景画像**: 「変形」以外の4シーン（お題提示・変形前・変形後・オチ）は静止画を用意し、軽いズーム（Ken Burns風）や切り替えで動きを出す
- **AI動画（変形シーンのみ）**: image-to-video系のAI動画生成で「変形」シーン（4秒程度想定）を作る。具体的なモデル・実行環境・コストは未検証（下記「変形シーンのAI動画生成（未検証）」参照）
- **AI音声**: ナレーション用TTS。エンジンは未定（`docs/REPO_RESEARCH.md` の比較を踏まえて選定、Codexレビューで確定させる）
- **テロップ**: ナレーションに合わせて表示。タイミング取得方法（自動 or 手動）は未定
- **効果音**: `assets/sfx/` から選択して合成（変形音、決めポーズ音など）
- **合成**: FFmpegベースを想定（静止画のズーム=zoompanフィルタ、切り替え=xfade等、変形シーンは生成済み動画クリップをそのまま合成）。Docker必須・重量ローカルモデル前提の構成は避ける（`docs/REPO_RESEARCH.md` 参照）

## 変形シーンのAI動画生成（未検証・要調査）

2026-07-14、ユーザーからのフィードバックにより「変形シーンは動画で見せたい（高品質にしたい）」という要望が判明し、当初の「いきなり重い動画生成AIは使わない」方針から、変形シーンに限定してAI動画生成を使う方針に変更した。

- 対象は「変形」シーン（4秒程度）のみ。他4シーンは静止画のまま
- 具体的なimage-to-videoモデル・ComfyUI/Flux環境との連携方法・必要VRAM・生成時間・RunPodでのコスト試算は**未検証**。存在確認できないモデル名・ノード名を断定しない
- RunPodのNetwork Volume 0GB運用（セッション毎に環境を再構築する前提）と、動画生成モデルのモデルサイズ・DL時間の相性を検証する必要がある
- 本実装に入る前に調査・Codexレビューを行う

## 表現面のルール

- 「トランスフォーマー」という名称を使わない
- 既存作品・既存キャラクターのデザイン・名称に寄せない。オリジナル変形ロボとして扱う
- 啓発系・注意喚起系のトーンにしない。エンタメ／妄想／図鑑／コメントお題型にする

## スコープ外（今回は扱わない）

- SNS自動投稿（後回し）
- 「変形」シーン以外へのtext-to-video/image-to-video動画生成の適用
- 実写・顔出し素材

## 決定事項（2026-07-13 Codexレビュー反映）

- テロップのタイミング取得方法: 初版は`episode.json`のシーン単位で手動秒指定とする（ASRによる自動抽出は将来差し替え候補）
- 機械可読データは`episodes/<話数>/episode.json`（標準JSON、追加ライブラリ不要）。人間向けの`plan.md`と併用する
- パイプラインは`validate`（マニフェスト検証）→`render`（動画合成）の2段構成とし、まず`validate`のみ実装する

## 未決定事項（実装前に確定させる）

- TTSエンジンの選定: Piperは日本語voice未提供＋GPLv3のため既定候補から除外。edge-ttsはオンライン依存のためoptional provider扱い。当面は事前生成済み音声ファイルを入力とする（詳細は`docs/CODEX_REVIEW_RULES.md`参照）
- AI画像生成の実行手段（既存のComfyUI/Flux環境を流用するか、別途軽量な手段を用意するか）
- 1本あたりのカット数・画像枚数の標準値（現状は5シーン=5カット想定）
- 変形シーンのimage-to-video生成手段（モデル・実行環境・VRAM要件・生成時間・コスト試算、いずれも未検証）
