# bokurobo（企画名: ぼくが考えた変形ロボ）

撮影なし・顔出しなしで、身近なモノ（動物・植物・建物・食べ物・日用品など）をオリジナルの変形ロボとしてAIでデザインし、15〜20秒のショート動画に仕上げる専用システムです。

## コンセプト

- 動物、植物、建物、食べ物、日用品などを題材に、オリジナルの変形ロボをAI画像として描く
- 「変形前 → 変形後」を見せる、図鑑的・妄想エンタメ的なショート動画にする
- AI背景画像 ＋ AI音声（ナレーション） ＋ テロップ ＋ 効果音 ＋ 軽いズーム/切り替えで構成する
- 撮影・顔出し・実写素材は使わない

## 重要方針

- 「トランスフォーマー」という名称は使わない
- 既存作品・既存キャラクターに寄せない。あくまでオリジナル変形ロボとして扱う
- 啓発系・注意喚起系のトーンは避ける（エンタメ／妄想／図鑑／コメントお題型）
- SNS自動投稿は後回し。最初はmp4生成までを対象にする
- いきなり重い動画生成AI（text-to-video系）は使わない。AI静止画＋音声＋テロップ＋効果音＋ズーム/切り替えの組み合わせで構成する
- `output/` はGit管理しない（生成物はローカル/RunPod側にのみ残す）
- `main` ブランチへの直接変更は禁止。作業はブランチを切って行う
- 実装に入る前に、Codexへagmsgでレビュー依頼する

## 想定実行環境

スマホ（Termux経由）・PC・RunPod（GPU、Network Volume 0GB運用でセッション毎に環境再構築）の3環境を横断で動かせることを前提に設計する。Docker必須構成や、セッション毎に重量モデルを再ダウンロードする構成は避ける方針。
