# bokurobo（企画名: ぼくが考えた変形ロボ）

撮影なし・顔出しなしで、身近なモノ（動物・植物・建物・食べ物・日用品など）をオリジナルの変形ロボとしてAIでデザインし、15〜20秒のショート動画に仕上げる専用システムです。

## コンセプト

- 動物、植物、建物、食べ物、日用品などを題材に、オリジナルの変形ロボをAI画像として描く
- 「変形前 → 変形後」を見せる、図鑑的・妄想エンタメ的なショート動画にする
- AI背景画像 ＋ AI音声（ナレーション） ＋ テロップ ＋ 効果音 ＋ ズーム/切り替え ＋ 変形シーンのみAI動画生成、で構成する（2026-07-14変更、下記「変形シーンの表現方針」参照）
- 撮影・顔出し・実写素材は使わない

## 重要方針

- 「トランスフォーマー」という名称は使わない
- 既存作品・既存キャラクターに寄せない。あくまでオリジナル変形ロボとして扱う
- 啓発系・注意喚起系のトーンは避ける（エンタメ／妄想／図鑑／コメントお題型）
- SNS自動投稿は後回し。最初はmp4生成までを対象にする
- AI動画生成は「変形」シーンのみに限定する。それ以外（お題提示・変形前・変形後・オチ）は静止画＋ズーム/切り替えで構成する（コスト・実装難易度を抑えるため。詳細は`docs/SPEC.md`参照）
- `output/` はGit管理しない（生成物はローカル/RunPod側にのみ残す）
- `main` ブランチへの直接変更は禁止。作業はブランチを切って行う
- 実装に入る前に、Codexへagmsgでレビュー依頼する

## 変形シーンの表現方針（2026-07-14決定）

当初は「いきなり重い動画生成AIは使わない」方針で、全シーンを静止画＋ズーム/切り替えのみで構成する予定だった。しかしユーザーからのフィードバックで「変形は動画で見せたい（高品質にしたい）」という要望が判明したため、方針を修正した。

- **変形シーンのみ**、text-to-video系のAI動画生成を使う（4秒程度を想定。2026-07-14時点で採用検討中のLoRAがtext-to-video専用のため、image-to-videoから変更。詳細は`docs/SPEC.md`参照）
- それ以外の4シーン（お題提示・変形前・変形後・オチ）は、コスト重視のため引き続き静止画＋ズーム/切り替え
- 具体的な動画生成モデル・実行環境（RunPodのGPU TIER要件、生成時間・コスト試算）は未検証。実装前に別途調査・Codexレビューを行う
- RunPodのNetwork Volume 0GB運用（セッション毎の環境再構築）との相性、コストへの影響を検証してから本実装に入る

## 想定実行環境

スマホ（Termux経由）・PC・RunPod（GPU、Network Volume 0GB運用でセッション毎に環境再構築）の3環境を横断で動かせることを前提に設計する。Docker必須構成や、セッション毎に重量モデルを再ダウンロードする構成は避ける方針。

## ディレクトリ構成

```
bokurobo/
├── README.md                 # このファイル
├── AGENTS.md                 # このリポジトリ専用のエージェント向けルール
├── docs/
│   ├── SPEC.md                # 動画仕様（尺・構成要素・技術方針）
│   ├── WORKFLOW.md            # 企画→生成→出力の作業フロー
│   ├── REPO_RESEARCH.md       # 類似OSS調査と採用/不採用の整理
│   └── CODEX_REVIEW_RULES.md  # Codexへのレビュー依頼運用ルール
├── episodes/                  # 1本ごとの企画データ（お題・台本・素材メモ）
│   └── _template/              # 新規エピソード作成用テンプレート
├── assets/                    # 固定素材（Git管理）
│   ├── sfx/                    # 効果音
│   ├── fonts/                  # テロップ用フォント
│   └── bgm/                    # BGM
├── src/                       # 生成パイプラインのコード（bokuroboパッケージ）
├── tests/                     # unittestによるテスト
└── output/                    # 生成物（mp4等）※Git管理外
```

## セットアップ

```
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
```

## 実行

```
# episode.json の検証
python -m bokurobo.cli validate episodes/<話数>/episode.json

# 動画の合成（要 ffmpeg/ffprobe）
python -m bokurobo.cli render episodes/<話数>/episode.json
```

## テスト

```
python -m unittest discover -s tests
```

## 状態

`episode.json`マニフェストのvalidatorと`render`（FFmpegベースの動画合成）を実装済み。詳細は `docs/SPEC.md` と `docs/WORKFLOW.md` を参照。TTSエンジン・変形シーンのAI動画生成モデルなど、生成AI周りの実行環境は未確定・未実装。
